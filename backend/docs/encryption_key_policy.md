# 端到端加密与密钥管理策略

## 加密范围

| 数据 | 加密方式 | 说明 |
|------|----------|------|
| 合同原文 (raw_text) | 存储加密 | 入库前 Fernet 加密，读取后解密 |
| 解析结果/中间文本 | 同原文 | 与 raw_text 一并加密存储 |
| 导出报告 | 可选文件加密 | 存盘时可加密；下载时解密或临时链接 |
| RAG 向量/片段 | 索引可加密 | 敏感片段加密后再建索引 |

## 密钥管理

- **主密钥**：通过环境变量 `ENCRYPTION_KEY` 注入（生产必配）；未设置时由 `JWT_SECRET` 派生（仅开发）。
- **密钥轮换**：建议每 `key_rotation_days`（默认 90 天）轮换；轮换时用新密钥加密新数据，旧数据可逐步重加密或保留至过期。
- **托管平台**：生产环境使用云厂商 KMS/Secret 管理 `ENCRYPTION_KEY`，禁止写入代码或配置文件。

## 配置清单

- `ENCRYPTION_KEY`: 可选，32 字节及以上密钥材料（或由 KMS 提供）。
- `ENCRYPTION_ENABLED`: 是否启用存储加密（默认 true）。
- `KEY_ROTATION_DAYS`: 密钥轮换周期（天）。
- `DATA_RETENTION_DAYS`: 数据保留天数，到期自动清理（默认 365）。
